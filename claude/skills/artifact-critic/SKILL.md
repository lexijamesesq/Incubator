# /artifact-critic — Structural and Voice Conformance Check

Check the specified artifact against structural, voice, and rating calibration criteria. Report deviations only.

## Invocation

```
/artifact-critic [idea-file-path]
```

- Required argument: path to the idea card file to check (e.g., `Ideas/authentic-assessment-rubric-design.md`)
- If no argument provided: report "No artifact path provided. Usage: /artifact-critic [idea-file-path]" and exit.

## Execution

1. Read the artifact file at `$ARGUMENTS`
2. Read the format example: `Examples/developing-card.md`
3. Read the voice guide: `persona.md`

## Format Spec (TL;DR Card)

Check structural conformance against this spec:

```
**Core insight:** One sentence — confident and declarative
**Problem it addresses:** One sentence — what's broken today
**Who cares:** One sentence — audiences and their stakes, comma-separated
**Strategic connection:** One sentence — name the priority and the mechanism

### Opportunity Assessment
| Dimension | Rating | Rationale |
Each rationale cell: One sentence — the judgment, not the evidence
No inline data points (N-values, market figures, survey sizes) in rationale cells

### Research Summary
3-5 bullets
Each bullet ends with a linked parenthetical source: ([Source Name](url or doc path))
Cross-domain signals condensed into exactly one bullet prefixed "Cross-domain:"

### Thought Outline
One paragraph — conceptual shape and strategic leverage, not a research recap

### Buildable Surface (optional — present only on principle-shaped ideas)
[generated by /buildable-surface]
If present, check:
- 3-5 numbered items, each one sentence
- No sub-labels (no "Feature candidates:", "Recommended surface:", or "Rationale:")
- If borderline flag present, appears on line after heading
If absent, this is expected (feature-shaped idea) — do not flag.

### Open Questions
Each item: a question only — no explanatory follow-up sentences after the question mark
```

Apply your system-level checking instructions against this spec, the format example, and the voice guide. Report deviations only using the format defined in your system instructions.
